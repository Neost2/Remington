Business Need
Patients in rural/low-income areas miss critical appointments (dialysis, oncology, cardiology, post-surgical) due to unreliable transportation — not lack of willingness. Validated across 15 stakeholder interviews (13 strong signals): patients, caregivers/RNs/CNAs, drivers, and clinic staff all lack a shared coordination and fallback system, causing no-shows, delayed treatment, and avoidable ER visits.
Project Objective
Deliver a working, demo-ready web app (API + mobile-first UI) by Aug 13, 2026 proving the full ride coordination loop — intake → match/assign → confirm → complete → survey — with automatic fallback escalation, using real auth and seeded demo data.

Constraints
  • Fixed deadline: Demo Day Aug 13, 2026
  • 3-person team, no dedicated QA/security/DevOps role
  • Must build on inherited schema, auth, and design system
  • Free/low-cost infra only (Amplify, Postgres, Twilio trial)
  • Weekly pass/fail milestone gates — slip means cut scope
  • No real PHI storage permitted
  • Demo must run on mobile device / unreliable Wi-Fi (seeded fallback required)
Assumptions
  • Cohort dates & Demo Day will not change
  • Amplify + Postgres hosting stays available at no unplanned cost
  • Twilio trial credentials suffice for demo-scale SMS
